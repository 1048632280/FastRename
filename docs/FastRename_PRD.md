# 极速文件重命名工具 - 产品开发需求文档 (PRD)

**版本：** v1.0  
**状态：** 待开发  

---

## 📌 项目背景与定位
传统操作系统中的重命名操作（如慢速双击或 F2 快捷键）在需要频繁、精确修改文件名局部内容时，存在操作繁琐、易误触的问题。本项目旨在开发一款轻量级的桌面端本地文件管理工具，主打**“无缝、精准、沉浸式”**的局部文本编辑体验，将修改文件名的操作简化至如同在纯文本编辑器中打字一般顺滑。

---

## 一、 核心交互设计（“无缝编辑”流）
本工具抛弃传统的“选中文件 -> 触发重命名状态 -> 全选文本”逻辑，采用“所点即所得”的极简交互路径。

### 1. 光标精准落点
*   **触发机制：** 鼠标单击文件列表中任意文件名的文字区域，立即进入编辑状态。
*   **光标定位：** 光标必须精准闪烁在用户鼠标点击的具体字符之间。例如，文件名“Report2023.docx”，用户点击“t”和“2”之间，光标直接在此处激活，**绝对禁止默认全选整个文件名**。

### 2. 全键盘驱动流 (Keyboard Flow)
为了保证用户在连续修改时不被迫在鼠标和键盘之间频繁切换，定义以下快捷键响应逻辑：

| 按键 | 当前状态 | 触发行为 |
| :--- | :--- | :--- |
| `Enter` (回车) | 正在编辑某文件 | 保存当前修改，并**自动将焦点和光标移动到下一个文件**，使其直接进入编辑状态。 |
| `Tab` | 正在编辑某文件 | 保存当前修改，并切换到下一个文件（同 Enter）。Shift+Tab 切换到上一个文件。 |
| `↑` / `↓` (方向键) | 列表选中或编辑中 | 在垂直方向的文件列表中切换焦点。若当前处于编辑状态，按下 `↑`/`↓` 保存并跳至上下相邻文件的对应光标位置。 |
| `Esc` (退出) | 正在编辑某文件 | 放弃当前输入的所有未保存字符，立即恢复文件的原始名称，并退出编辑状态。 |

### 3. 静默保存机制 (Auto-Save on Blur)
*   **失去焦点即保存：** 当用户点击列表的空白处，或者使用鼠标直接点击另一个文件时，上一个被编辑的文件自动静默保存（触发重命名指令），**严禁出现任何“是否保存修改”的二次确认弹窗**。

---

## 二、 扩展名（后缀）保护机制
由于工具的核心场景是高频修改，极易发生误删文件扩展名导致文件不可用的问题，必须从界面布局上进行物理隔离。

### 1. 分离式栏目设计（强烈推荐）
*   **界面布局：** 在列表视图中，强制将“主文件名”与“扩展名（类型）”拆分为两列独立显示。
*   **编辑限制：** 用户在界面上点击操作时，**只能触发第一列“主文件名”的编辑状态**。第二列“扩展名”区域设置为只读（或双击才可修改的特殊保护状态）。
*   **视觉呈现：** 文件列表中不再显示最后的点号（如 `.pdf`）。第一列显示 `项目计划书`，第二列显示 `pdf文件` 或 `pdf`。

---

## 三、 防错与安全网（容错机制）
文件系统具有强规则限制，且由于采用静默即时保存逻辑，必须配套严密的防护和撤回功能。

### 1. 即时非法字符拦截
*   **规则：** 操作系统严禁文件名包含以下字符： `\ / : * ? " < > |`
*   **拦截表现：** 当工具处于编辑状态，且用户在键盘上按下了上述非法字符键时，输入框内**不产生该字符输入**，同时输入框边框闪烁红色警示，或在鼠标位置弹出极简的悬浮文字提示（1秒后自动消失）。
*   **优势：** 将报错前置到“输入时”，而不是等到用户按下回车“保存时”才阻断流程。

### 2. 重名冲突智能处理
*   **实时检测：** 在用户输入过程中，后台实时比对同目录下的其他文件名。如果输入的名字已存在，输入框文字或边框变为橙色预警。
*   **静默处理：** 若用户无视预警强制保存（按下Enter或移开鼠标），系统自动在文件名末尾追加编号。例如：若 `测试报告` 已存在，则自动保存为 `测试报告(1)`，避免因报错弹窗打断用户的连续操作心流。

### 3. 全局撤销与时光机 (Ctrl+Z / Cmd+Z)
*   **功能定义：** 系统需在内存中维护一份“重命名操作历史栈”。
*   **操作逻辑：** 用户随时按下 `Ctrl+Z`（Mac下为 `Cmd+Z`），不仅能撤销输入框内的文字输入，更能在文件已经真实更名并落盘后，**直接在硬盘层面将最后一次改名的文件恢复原样**。
*   **连续撤销：** 支持连续多次按下撤销快捷键，依次恢复倒序的重命名操作。

---

## 四、 系统执行策略（保存时机）

> **💡 底层逻辑说明：** 本工具采用**“即时生效 (Real-time Execution)”**策略，不提供“批量应用”或“确认修改”按钮。
> 
> 用户的每一次“保存”动作（按下Enter、点击其他区域导致失去焦点），系统都会在后台**立即调用操作系统的 File Rename API 真实修改硬盘上的文件**。此策略最大化降低了用户的认知负担和操作步骤，配合上述第三章的“全局撤销”功能，足以保障数据安全性。
